# Contribuicao

Este projeto e academico e individual, mas este guia registra como futuras contribuicoes poderiam ser organizadas.

## Padrao de Branches

- `main`: versao principal do projeto.
- `feature/nome-da-feature`: novas funcionalidades.
- `docs/nome-da-documentacao`: ajustes de documentacao.
- `fix/nome-do-ajuste`: correcoes.

## Padrao de Commits

Use mensagens curtas e objetivas, preferencialmente no estilo Conventional Commits:

```text
docs: adiciona diagrama de classes
feat: cria fluxo de pedidos
fix: corrige regra de cancelamento
```

## Revisao

Antes de entregar ou publicar:

- Conferir se o README esta atualizado.
- Conferir se os diagramas PlantUML abrem corretamente.
- Conferir se as regras de negocio estao coerentes com os requisitos.
- Conferir se nao ha informacoes sensiveis no repositorio.
